# Session Goal

---

# Decisions Made

## Project Identity

DragonBoard is officially defined as:

> A companion tool for Dragonbane.

DragonBoard is **not** intended to replace Foundry VTT or automate Dragonbane gameplay.

Its purpose is to assist the Game Master while preserving the tabletop experience.

---

## Gameplay Philosophy

Physical components remain part of the game.

The following components will **not** be digitized:

* Dice
* Initiative Deck
* Treasure Deck
* Adventure Cards
* Improvised Weapon Cards

DragonBoard only manages the digital components that improve gameplay.

---

## Development Philosophy

The project follows these principles:

* Mobile-first development
* Tablet-first gameplay
* Dragonbane-first design
* Free Forever
* Simplicity over Features
* The 10 Second Rule

# Core Domain Model

The following core objects were finalized.

* Campaign
* Map
* Token
* Fog
* Journal
* Assets
* Settings
* GameState
* Camera
* Condition

Important decisions:

* HP belongs to Token.
* Position belongs to Token.
* Fog belongs to Map.
* Character entity intentionally omitted.
* Every object belongs to a Campaign.
* GameState stores runtime data only.
